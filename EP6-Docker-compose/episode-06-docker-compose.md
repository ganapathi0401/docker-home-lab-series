# Episode 6: Docker Compose

## Interview Questions & Answers

### Q1: What is the core idea behind docker compose?
**Answer:** It focuses on running applications in isolated, lightweight containers to ensure consistency across environments.

### Q2: Why is this important in real-world Docker setups?
**Answer:** Poor understanding of docker compose leads to fragile containers, security risks, and operational issues in production.

### Q3: Common mistakes engineers make?
**Answer:** Treating containers like VMs, ignoring immutability, and mismanaging resources or networking.

### Q4: How would you explain this to a junior engineer?
**Answer:** Start with the problem Docker solves, then explain how containers simplify deployment and debugging.

### Q5: Give a real production example.
**Answer:** A container crash caused by missing volumes or incorrect environment variables leading to downtime.
